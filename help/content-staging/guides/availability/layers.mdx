Appointments Booking uses a **layered availability engine**. Multiple layers combine to determine the final availability for any bookable entity at any point in time.

## How Layers Work

Layers are evaluated in priority order. Each layer can:
- **Allow** time slots (e.g., opening hours)
- **Block** time slots (e.g., maintenance windows, holidays)
- **Override** lower-priority layers

## Default Layers

| Layer | Priority | Purpose |
|-------|----------|---------|
| **Opening Hours** | Base | Weekly recurring schedule |
| **Existing Bookings** | High | Blocks already-booked slots |

## Custom Layers (via Addons)

Addons can register custom availability layers. For example, a facility booking addon might add a "Maintenance Block" layer that blocks court availability during scheduled maintenance.

See the [Developer Docs](/developers/architecture/) for the `AvailabilityLayerRegistry` API.
